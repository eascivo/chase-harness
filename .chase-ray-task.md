# Chase Ray — 多项目常驻编排器

## 背景

Chase 是一个零依赖纯 Python 的多 Agent 自主开发框架（Planner→Negotiator→Generator→Evaluator），当前只支持单项目串行执行。

现在需要扩展 Chase，新增 `ray` 子命令，使其成为**多项目常驻编排器**，能同时管理多个 Chase 项目，支持优先级调度、依赖管理、动态派发。

## 核心需求

### 1. CLI 子命令

```
chase ray init              # 在当前目录创建 RAYSPACE.md 模板
chase ray start [--daemon]  # 启动编排循环（前台或守护进程）
chase ray dispatch <path> [--priority N] [--depends-on A,B]  # 动态派发新项目
chase ray status            # 查看所有项目状态汇总
chase ray pause <name>      # 暂停某项目
chase ray resume <name>     # 恢复某项目
chase ray priority <name> <N>  # 调整优先级
chase ray stop              # 优雅停机
chase ray remove <name>     # 移除项目（不删文件）
```

### 2. RAYSPACE.md 格式

```markdown
# Chase Ray — 多项目编排

max_parallel: 2
log_dir: .chase-ray/logs

## 项目队列

| name | path | priority | depends_on | status |
|------|------|----------|------------|--------|
| ai-cs-system | ~/Project/ai-cs-system | 0 | | pending |
| research-system | ~/Project/research-system | 1 | | pending |
| marketing-pilot | ~/Project/marketing-pilot | 2 | ai-cs-system | blocked |
```

也可以用 JSON：`.chase-ray/queue.json`

```json
{
  "max_parallel": 2,
  "projects": [
    {"name": "ai-cs-system", "path": "~/Project/ai-cs-system", "priority": 0, "depends_on": [], "status": "pending"},
    {"name": "research-system", "path": "~/Project/research-system", "priority": 1, "depends_on": [], "status": "pending"},
    {"name": "marketing-pilot", "path": "~/Project/marketing-pilot", "priority": 2, "depends_on": ["ai-cs-system"], "status": "blocked"}
  ]
}
```

### 3. 常驻编排逻辑

```
while running:
  1. 扫描 queue.json
  2. 按优先级排序，拓扑检查依赖
  3. 取 max_parallel 个可执行项目
  4. 对每个项目：subprocess 起 chase run --workspace <path>
  5. 轮询子进程状态（非阻塞）
  6. 子进程完成 → 更新状态 → 检查依赖释放 → 补位启动
  7. 全部完成或收到 stop 信号 → 优雅退出
```

状态流转：`pending → running → completed / failed / paused → (resume → running)`

### 4. 守护进程模式

- `chase ray start --daemon` → fork 到后台，写 PID 到 `.chase-ray/ray.pid`
- 支持信号：SIGTERM 优雅停机（等当前 Chase 完成），SIGUSR1 重新加载 queue
- 提供 macOS launchd plist 模板：`chase ray launchd --generate`

### 5. 外部派发 API（文件协议）

Hermes 或其他工具可以直接写 queue.json 来派发项目，Ray 会自动检测变化：

```bash
# 等价于 chase ray dispatch
python3 -c "
import json
q = json.load(open('.chase-ray/queue.json'))
q['projects'].append({'name':'new-proj','path':'~/Project/new-proj','priority':0,'depends_on':[],'status':'pending'})
json.dump(q, open('.chase-ray/queue.json','w'), indent=2)
"
```

Ray 每 5 秒 poll 一次 queue.json，检测到新项目自动调度。

## 技术约束

- **零外部依赖** — 只用 Python 标准库（subprocess, json, pathlib, signal, os, time, argparse, logging）
- **兼容现有 Chase** — 不改动 chase/orchestrator.py 等现有代码，Ray 只调用 `chase run` 子进程
- **Python 3.11+** 兼容
- 新文件全部放在 `chase/ray/` 目录下


## 验收标准

1. `chase ray init` 生成 RAYSPACE.md 和 .chase-ray/queue.json
2. `chase ray status` 显示所有项目状态表格
3. `chase ray dispatch` 能动态添加项目到队列
4. `chase ray start` 能同时跑多个 chase run 子进程
5. 优先级 P0 > P1 > P2 正确排序
6. depends_on 阻塞正确生效（被依赖项目完成后才启动）
7. `chase ray pause/resume` 能暂停/恢复子进程
8. `chase ray stop` 优雅停机
9. daemon 模式正常运行，PID 文件正确
10. 不引入任何外部 pip 依赖
