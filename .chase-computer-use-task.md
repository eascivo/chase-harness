# 任务：Chase Generator Computer Use 能力

## 目标
为 Chase 的 Generator Agent 增加 Computer Use（浏览器自动化测试）能力，让 Generator 在实现代码后能自动操控浏览器验证功能是否正常。

## 背景
Chase 是一个零依赖纯 Python 的多 Agent 自主开发框架，位于 ~/Project/claude-harness/chase/。
当前 Generator 通过 subprocess 调用 `claude -p` 执行代码生成，但无法验证 UI/Web 功能。

## 具体要求

### 1. 新增 chase/computer_use.py
- 封装浏览器自动化能力，基于 CDP（Chrome DevTools Protocol）连接本地 Brave/Chrome
- 使用 Python 标准 urllib/http.client（零依赖原则），不引入 playwright/selenium
- 核心功能：
  - `launch_browser(url)` — 启动浏览器打开页面
  - `screenshot(path)` — 截图保存
  - `click(selector)` — 点击元素
  - `type_text(selector, text)` — 输入文本
  - `get_page_content()` — 获取页面文本内容
  - `evaluate_js(script)` — 执行 JavaScript
  - `close_browser()` — 关闭浏览器

### 2. 修改 chase/agents/generator.py
- Generator 完成代码后，检查 negotiated contract 中是否包含 "UI" / "web" / "browser" / "visual" 等关键词
- 如果是 Web 相关的 sprint，自动调用 computer_use 进行浏览器验证
- 将截图和页面内容作为 Evaluator 的额外输入

### 3. 修改 chase/agents/evaluator.py  
- Evaluator 接收可选的 browser_evidence 参数（截图路径 + 页面内容）
- 评估时如果 browser_evidence 存在，加入评估依据

### 4. Generator Prompt 增强
- 在 generator prompt 中加入 Computer Use 工具说明
- 告诉 Generator 可以调用浏览器工具来验证实现

## 约束
- **零额外依赖** — 只用 Python 标准库（urllib, json, subprocess）
- CDP 连接使用 http://localhost:9222（Brave 默认远程调试端口）
- 不改动现有的 Planner/Negotiator 流程
- 保持与现有 `.chase/` 目录结构的兼容性

## 测试
- 手动验证：确保 `python3 -c "from chase.computer_use import launch_browser; ..."` 能连接到 Brave CDP
- 不需要写单元测试（CDP 需要真实浏览器）
