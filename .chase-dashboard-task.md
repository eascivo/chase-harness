# 任务：Chase Ray Web Dashboard

## 目标
为 Chase Ray 多项目编排器增加一个轻量 Web Dashboard，实时显示所有项目的进度、状态和资源消耗。

## 背景
Chase Ray 是 Chase 的多项目常驻编排器，位于 ~/Project/claude-harness/chase/ray/。
当前只能通过 CLI `chase ray status` 查看状态，需要一个 Web 界面。

## 具体要求

### 1. 新增 chase/ray/dashboard.py
- 纯 Python 标准库实现 HTTP 服务器（http.server），零额外依赖
- 端口默认 8765，可通过 `--port` 参数配置
- 提供 WebSocket 推送实现实时刷新（使用标准库的 asyncio + websockets... 不对，websockets 不是标准库）
- 改用 **Server-Sent Events (SSE)** 实现实时推送（纯 HTTP，零依赖）
- 提供 REST API + 静态 HTML 页面

### 2. REST API 端点
- `GET /api/status` — 返回所有项目状态 JSON（读取 queue.json + 各项目 .chase/ 状态）
- `GET /api/project/{name}` — 返回单个项目详情（sprint 进度、cost、eval scores）
- `GET /api/events` — SSE 端点，每 5 秒推送状态变更
- `POST /api/dispatch` — 动态派发新项目（等同于 CLI dispatch）
- `POST /api/priority` — 修改优先级

### 3. 静态 HTML Dashboard（内嵌在 Python 里）
- 深色主题，和 Hermes 风格一致（#1a1a2e 背景，#00d4ff 强调色）
- 顶部：Ray 状态（运行时间、max_parallel、总 cost）
- 项目卡片列表：
  - 项目名、优先级标签（P0/P1/P2 彩色）、状态指示灯
  - Sprint 进度条（已完成/总数）
  - 当前阶段（Planner/Negotiator/Generator/Evaluator）
  - 累计 cost
  - 最新 eval score（彩色显示，绿色=1.0，黄色=0.7+，红色<0.7）
- SSE 自动刷新，无需手动刷新页面
- 响应式布局，手机也能看

### 4. 集成到 chase/ray/cli.py
- `chase ray dashboard` — 启动 Web Dashboard（前台，可用于调试）
- `chase ray start --dashboard` — 启动 Ray 时同时启动 Dashboard（后台线程）

### 5. 集成到 chase/ray/daemon.py
- Ray 守护进程启动时，如果 `--dashboard` 开启，在单独线程启动 HTTP server
- Dashboard 作为 Ray 的子组件，Ray stop 时自动关闭

## 设计参考
- 状态数据来源：`.chase-ray/queue.json` + 各项目 `.chase/sprints/` + `.chase/logs/cost-tracking.json`
- HTML 内嵌在 dashboard.py 的字符串常量中，不需要额外文件
- SSE 格式：`data: {json}\n\n`

## 约束
- **零额外依赖** — 只用 Python 标准库（http.server, json, threading, os, pathlib）
- 不引入 Flask/FastAPI/任何第三方库
- HTML/CSS/JS 全部内嵌，不产生额外文件
- 不改动现有的 Ray 调度逻辑

## 测试
- 启动后访问 http://localhost:8765 能看到 Dashboard 页面
- GET /api/status 返回正确的 JSON
