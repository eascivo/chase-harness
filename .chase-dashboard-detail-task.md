# 任务：Chase Ray Dashboard 项目详情页

## 目标
增强现有的 Chase Ray Web Dashboard，增加单个项目详情视图，点击项目卡片可以查看 MISSION 目标、Sprint 计划、验收标准、测试结果、评估详情等。

## 背景
Chase Ray Dashboard 已实现基本概览页（chase/ray/dashboard.py），位于 ~/Project/chase-harness/chase/ray/dashboard.py。
当前有 GET /api/status 和 GET /api/project/{name} 端点，但前端只有概览卡片，没有详情页。

## 现有 API 返回数据
GET /api/project/{name} 已返回：
```json
{
  "name": "...", "status": "running", "priority": 0,
  "sprint_completed": 3, "sprint_total": 6,
  "current_stage": "Generator", "cost": 5.23, "eval_score": 1.0,
  "sprints": [
    {"id": "1", "stage": "completed", "eval_score": 1.0, "cost": 2.5},
    ...
  ]
}
```

## 需要新增的内容

### 1. 扩展 GET /api/project/{name} API
读取更多项目详情数据：
- **MISSION.md 内容** — 从项目根目录读取 MISSION.md（或 .chase/mission.md）
- **每个 Sprint 的详情**：
  - `contract.md` — Sprint 原始目标（从 .chase/sprints/XX-contract.md 读取）
  - `negotiated.md` — 验收标准（从 .chase/sprints/XX-negotiated.md 读取）
  - `result.md` — Generator 输出摘要（从 .chase/sprints/XX-result.md 读取）
  - `eval.json` — 评估结果（从 .chase/sprints/XX-eval.json 读取）
- **cost-tracking.json** — 各阶段费用明细

API 返回示例：
```json
{
  "name": "ai-cs-system",
  "mission": "# 目标\n将 ai-cs-system 从 MVP...",
  "sprints": [
    {
      "id": "1",
      "stage": "completed",
      "contract": "## Sprint 1: 数据库模块重构\n...",
      "negotiated": "## 验收标准\n1. SQLite DatabaseManager 实现\n...",
      "result": "Generator output summary...",
      "eval": {"score": 1.0, "verdict": "PASS", "feedback": "...", "criteria": [...]},
      "cost": 2.5
    }
  ],
  "total_cost": 15.3
}
```

### 2. 前端详情页
- 点击概览页的项目卡片 → 切换到详情视图（单页应用，JS 路由，不用刷新）
- 详情页布局：
  - **顶部**：项目名 + 状态 + 优先级 + 返回按钮
  - **MISSION 区块**：渲染 MISSION.md 内容（支持基本 markdown：# ## ** ` 代码块）
  - **Sprint 时间线**：垂直时间线，每个 Sprint 一个节点
    - 节点头部：Sprint 编号 + 状态（✅/🔄/❌）+ 评估分数 + cost
    - **可展开/折叠**的 Sprint 详情：
      - 📋 Contract（原始目标）
      - ✅ Negotiated（验收标准，逐条显示，通过的绿色标记）
      - 🔨 Result（Generator 输出，截断显示，可展开全文）
      - 📊 Evaluation（分数 + verdict + feedback）
  - **底部**：总 cost 汇总

### 3. Markdown 渲染
- 纯 JS 实现简易 Markdown 渲染（不引入外部库）
- 支持：`#` `##` `###` `**` `` ` `` ` ``` ` `- ` 列表 `1.` 有序列表
- 代码块带深色背景

### 4. 交互优化
- Sprint 详情默认折叠，点击展开（避免信息过载）
- 返回按钮回到概览页
- 加载中状态（Spinner）
- 页面标题动态更新

### 5. 概览页卡片增加点击样式
- hover 效果 + cursor: pointer
- 卡片上显示当前 Sprint 编号
